Say hi to Serializer!

Problem:

someElement.innerHTML = someElement.innerHTML;
// Alert: This line above deletes all someElement's children nodes as it overrides its html content

This is very inconvenient for example you want to save the selection range on the DOM object and restore it later on but you can't because the container elements assigned to the saved range don't exist anymore after you have overrided the object's innerHTML.

Solution: 

var serializer = new Serializer(editor); // editor is a root node

The mechanism of Serializer is with the given root node, in this example, editor, it saves the position of a specific node and serialize it into a string. Therefore, when the root node innerHTML is overriden, it can still find that specific node through the saved string.
